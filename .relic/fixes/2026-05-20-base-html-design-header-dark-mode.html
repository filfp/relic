Fix: base.html — Design, Header & Dark Mode

ID: 2026-05-20-base-html-design-header-dark-mode Date: 2026-05-20 Spec: 008-html-spec-mode Status: solved Classification: code-bug

Issue

The current base.html output is visually broken and lacks the features needed to make spec HTML files useful as first-class artefacts. Three problems were observed:

  1. Broken comment block — the <!-- RELIC COMPONENTS --> block contains Mermaid-style --> arrow strings (e.g. init --> config) that prematurely terminate the HTML comment. The browser renders the example diagram and "CSS UTILITIES" text as visible page content at the very top of the file.
  2. No navigation header — there is no sticky header with a dark mode toggle or quick links to open spec.md, plan.md, tasks.md directly from the rendered HTML.
  3. Weak visual design — the base template is too sparse to guide the LLM toward rich, synthesised output. The result looks like a plain HTML dump rather than a polished spec view.
⚠ The comment termination bug means that every spec and fix HTML file scaffolded in html mode currently renders broken content at the top of the page. This affects all users in html mode.

Root Cause

✕ code-bug — Implementation error in templates/base.html; the spec and contracts are correct.

Bug 1 — HTML comment termination

HTML comments end at the first occurrence of -->. The RELIC COMPONENTS inventory uses Mermaid-style flow syntax in its examples:

graph TD
init --> config
config -->|html mode| scaffold

Each --> in those examples terminates the surrounding <!-- ... --> comment. Everything after the first edge arrow becomes raw HTML in the DOM — visible to the user as text and as a rendered <relic-flow> element.

Bug 2 — No header / dark mode / nav links

These were not specified in FR-18 through FR-22 of spec 008. They are implicit in FR-15 ("enrichment, richer structure") but were not implemented in the base template, leaving every LLM-generated spec file lacking basic navigation and theming infrastructure.

Bug 3 — LLM synthesis guidance too weak

The prompt templates instruct the LLM to "update the spec HTML", but do not explicitly prohibit mechanical transcription. The result is copy-paste content rather than synthesised visualisations.

Proposed Changes

1 — Fix the RELIC COMPONENTS documentation block

Replace the HTML comment with a <template id="relic-docs"> element (or a <script type="text/plain"> block). Templates are inert — they are never rendered or parsed as HTML, so --> sequences are safe inside them. Update the component inventory to use a <template> wrapper.

ℹ Alternative: escape all --> to -→ or → inside the comment block. The template approach is cleaner and eliminates the class of bug entirely.

2 — Add a sticky header with dark mode toggle and nav links

Add to base.html:

  • A <header> element, sticky at top, with spec ID + title on the left
  • A dark mode toggle button (moon/sun icon, no external asset — pure Unicode or inline SVG)
  • Three icon links: spec.md, plan.md, tasks.md — using relative file:// paths so they open in the browser's default viewer. Each link uses a small emoji/icon (📄 or ≡) and the filename as a tooltip.
  • CSS variables (--bg, --surface, --text, --border, etc.) that the dark mode toggle flips via a data-theme="dark" attribute on <html>

3 — Refine overall visual design

Update the CSS in base.html:

  • Switch all hardcoded colours to CSS custom properties for theme-ability
  • Constrain body max-width to ~900px with centered layout for readability
  • Add a subtle left sidebar / table-of-contents generated from <h2> headings via JS
  • Improve section spacing, typography scale, and table striping
  • Add a <relic-chip> component for compact inline metadata tags

4 — Strengthen LLM synthesis guidance in prompt templates

Update the HTML step in specify.md, clarify.md, plan.md, tasks.md, implement.md, fix.md, solve.md to add explicit anti-transcription language:

ℹ Proposed addition: "Do NOT copy Markdown text verbatim into the HTML. Each section must synthesise the information into the most appropriate visual form: a flow diagram instead of a bullet list, a table instead of prose, a status badge instead of a heading. If a section would be identical to the Markdown, you are doing it wrong."
