Fix: relic use --fix ignores html mode

ID: 2026-05-20-use-fix-html-extension Date: 2026-05-20 Spec: 003-fix-solve-workflow Status: solved Classification: code-bug

Issue

Running relic use --fix <fix-id> always looks for .relic/fixes/<fix-id>.md regardless of the project's configured mode. When mode is "html", fix documents are created as .html files — but the command hard-codes the .md extension and reports a file-not-found error, leaving session.fix unset and blocking the entire fix pipeline.

Impact: In html mode, relic use --fix is completely broken. The user cannot activate a fix — /relic.solve will refuse to proceed because current_fix is always null. The only workaround is to skip fix activation and proceed directly with /relic.solve while the fix document path is still in context.

Observed error

$ relic use --fix 2026-05-20-base-html-design-header-dark-mode
Fix document not found: .relic/fixes/2026-05-20-base-html-design-header-dark-mode.md
Run /relic.fix first to create a fix document.

The file .relic/fixes/2026-05-20-base-html-design-header-dark-mode.html exists and is valid, but use.ts never checks for it.

Root Cause

code-bug — Implementation error in packages/core/src/commands/use.ts. The spec (003) correctly states that fix documents follow the project mode; the implementation never reads mode.

Faulty code — use.ts lines 32–40

// --fix branch
if (options.fix) {
  const fixDoc = join(relicDir, "fixes", options.fix + ".md");  // ← hardcoded .md
  if (!fileExists(fixDoc)) {
    console.error(`Fix document not found: .relic/fixes/${options.fix}.md`);
    console.error("Run /relic.fix first to create a fix document.");
    process.exit(1);
  }
  writeSession(relicDir, { ...readSession(relicDir), fix: options.fix });
  console.log(`Active fix: ${options.fix}`);
  return;
}

The branch constructs the path with + ".md" and never consults config.json. The readMode helper already exists in @relic/utility — it simply was not wired up here. Note that relic use --clear-fix works correctly; only the --fix branch is affected.

graph LR use[relic use --fix] mode{check mode} md[look for .md] html[look for .html] write[write session.fix] err[error: not found] use --> mode mode -->|md| md mode -->|html| html md -->|exists| write html -->|exists| write md -->|missing| err html -->|missing| err

Current implementation skips the mode check and always takes the md branch.

Proposed Changes

Code change — packages/core/src/commands/use.ts

Import readMode from @relic/utility; derive the file extension from it; update the error message to name the correct extension. No other files need changing — session.ts, context.ts, and fix.ts all store and read the bare fix ID; the extension is resolved only at lookup time.

Diff (logical)

// Before
import { join } from "path";
import { fileExists, dirExists, readSession, writeSession } from "@relic/utility";
import { availableSpecs } from "@relic/utility";

// After
import { join } from "path";
import { fileExists, dirExists, readSession, writeSession, readMode } from "@relic/utility";
import { availableSpecs } from "@relic/utility";

// --fix branch (before)
const fixDoc = join(relicDir, "fixes", options.fix + ".md");
if (!fileExists(fixDoc)) {
  console.error(`Fix document not found: .relic/fixes/${options.fix}.md`);
  ...
}

// --fix branch (after)
const ext = readMode(relicDir) === "html" ? ".html" : ".md";
const fixDoc = join(relicDir, "fixes", options.fix + ext);
if (!fileExists(fixDoc)) {
  console.error(`Fix document not found: .relic/fixes/${options.fix}${ext}`);
  ...
}

Resolved — 2026-05-20

Applied. One-line change in packages/core/src/commands/use.ts:

  • Added readMode to the import from @relic/utility.
  • Replaced options.fix + ".md" with options.fix + ext where ext = readMode(relicDir) === "html" ? ".html" : ".md".
  • Updated the error message to reference the correct extension.

Verified: ./dist/relic use --fix 2026-05-20-use-fix-html-extension now prints "Active fix: …" and relic context confirms current_fix is set correctly.
