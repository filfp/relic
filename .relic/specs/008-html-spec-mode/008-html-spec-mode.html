008-html-spec-mode HTML Spec Mode
≡ spec ⊞ plan ☑ tasks

Overview

draft spec-008 2026-05-19

Relic's Markdown-only output limits how richly AI agents can present spec information. This spec adds an HTML mode — when enabled, every workflow command gains a final HTML step where the LLM enriches a named spec file with charts, flow diagrams, status views, and structured cross-references that Markdown cannot express.

The token cost is kept low by base.html — a self-contained component library the LLM calls declaratively rather than authoring raw SVG or CSS from scratch.

What changes when you run relic mode html

graph LR Init[relic init] --> Config[config.json mode=html] Config --> Scaffold[relic scaffold] Scaffold --> SpecHTML[spec-id.html created] SpecHTML --> Workflow[specify / clarify / plan ...] Workflow --> HTMLStep[HTML step runs] HTMLStep --> Updated[spec-id.html updated]

Component Library (base.html)

Component docs live in <template id="relic-docs"> (not an HTML comment). HTML comments are broken by --> arrow syntax in flow diagram examples — a <template> element is inert and immune to this class of bug (FR-21).

Clarifications applied (2026-05-20)

The following requirements were added or corrected after the first implementation:

Scope

✓ In scope

  • Config infrastructure: project-config.ts, config.json
  • relic mode <md|html> CLI command
  • mode field in relic context
  • relic scaffold creates <spec-id>.html
  • templates/base.html — all components, header, dark mode
  • HTML step in 5 spec + 2 fix prompt templates
  • Anti-transcription rules (NFR-8)
  • relic validate for HTML files (FR-27 — blocked)

✕ Out of scope

  • Filesystem watching / auto-regeneration
  • Local HTTP server for HTML files
  • HTML for shared artifacts (domains, contracts…)
  • HTML for ask / scan / analyse commands
  • Build-step CSS frameworks
  • Changing or deleting Markdown files
