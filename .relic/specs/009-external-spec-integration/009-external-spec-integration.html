External Spec Integration

Spec: 009-external-spec-integration — 2026-05-20

Overview

draft

Enterprise teams keep a canonical spec repository (FRs, NFRs, User Stories, ADRs) outside the implementation repo. Today Relic's AI workflow commands operate without that context — plans are written without the FR that constrains them, specs scoped without the ADR that governs the architecture. This spec adds a first-class integration path: point Relic at the external folder via config.json, surface it in relic context, let each spec declare which external documents it depends on (external_reads in artifacts.json), and have every workflow command load those documents automatically.

Read-only integration — Relic never writes to the external repo. One external repo per project. Local filesystem only (no remote auth).

Requirements

Key functional requirements

Constraints

Path traversal outside external.specsDir via ../ is rejected at relic external link time — not deferred to validate. Cross-platform: path separator normalisation required for Windows compatibility. All new commands: JSON default, --text for human output.

Scope

In scope

  • config.json external block
  • relic external command (get / set / link / list)
  • relic context external field
  • artifacts.json external_reads array
  • relic validate warnings for missing external docs
  • 6 prompt templates: external_reads load step
  • project-config.ts helpers: readExternalSpecsDir / writeExternalSpecsDir

Out of scope

  • Writing to the external repo
  • Multiple external repos
  • Remote / authenticated access
  • Indexing external docs via relic search
  • Nested Relic sessions inside the external repo
