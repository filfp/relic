Overview

draft spec-009 2026-05-20

Enterprise teams keep a canonical spec repository (FRs, NFRs, ADRs) outside the implementation repo. Today Relic workflows run blind — plans written without the FR that constrains them, specs scoped without the ADR that governs the architecture.

This spec adds a first-class read-only integration path: point Relic at the external folder via config.json, surface it in every relic context response, and let each spec declare the exact external documents it depends on via external_reads in artifacts.json. Every workflow command then loads those documents automatically before doing any work.

Core constraint: Relic never writes to the external repo. One external repo per project. Local filesystem only — no remote auth, no network access.

Prompt template changes

Six prompt templates gain a new external_reads load step that fires before any other work: specify, clarify, plan, tasks, implement, fix.

Path traversal outside external.specsDir via ../ is rejected at relic external link time — not deferred to validate. Cross-platform path separator normalisation is required for Windows compatibility.

Scope

✓ In scope

  • config.json external block
  • relic external command (get / set / link / list)
  • relic context external field
  • artifacts.json external_reads array
  • relic validate warnings for missing external docs
  • 6 prompt templates: external_reads load step
  • project-config.ts: readExternalSpecsDir / writeExternalSpecsDir

✕ Out of scope

  • Writing to the external repo
  • Multiple external repos per project
  • Remote / authenticated repo access
  • Indexing external docs via relic search
  • Nested Relic sessions inside the external repo
